I

Project Notes

Socket Lifecycle, Carefully Implemented.

The server follows a deterministic flow: socket creation, bind, listen, accept, parse, route, and respond. Every stage has explicit error handling and RFC-aware validation.

It emphasizes predictable behavior under malformed requests and path traversal checks during filesystem resolution.

  • Request-Line Parser
  • Header Scanner
  • Status Dispatch
  • Static File I/O
II

Project Notes

RAII Meets Protocol Correctness.

Connection objects own socket lifetimes and transfer ownership to worker threads using move semantics. This approach avoids leaks under failure and keeps teardown explicit.

Persistent connections and chunked responses were tested with realistic client behavior to validate protocol compliance under parallel load.

  • Thread-per-Connection
  • Keep-Alive
  • Chunked Encoding
  • RAII Ownership
